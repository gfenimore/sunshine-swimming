Data Scrub — Prepare the CSV

45 minutes

Amanda's Google Sheet (Swimmer_Questionnaire___Sunshine_Swimming_Responses.xlsx) has 112 rows, 19 columns. HoneyBook's CSV import accepts: name, email, phone, address, notes. All other swimmer-specific data goes into the notes field as structured text.

Source columns → HoneyBook mapping Parent/Guardian Name(s) → Contact Name (REQUIRED) Email → Email (REQUIRED — min for import) Phone Number → Phone Home Address → Address Lesson location → Notes (append) Swimmer Name → Notes (append) Birthday → Notes (append) Comfort level → Notes (append) Medical conditions → Notes (append) Prior injuries → Notes (append) Additional swimmers → Notes (append) Preferred days/times → Notes (append) How did you hear → Notes (append) Anything else → Notes (append) DROP: Column "x" (timestamp), "Column 19" (empty)
⚠️ LOTR names → real names: The test data uses LOTR character names. For the production import, use Amanda's real client data. For the demo prototype, keep LOTR names — they make it obvious this is test data and prevent any accidental exposure of real client info.
📋 Data quality issues found in the spreadsheet:
• Phone numbers inconsistently formatted (some have dots, some have dashes, some are raw numbers)
• Some addresses missing city/zip
• Duplicate entries for siblings (e.g., "noah & navy clerfond" in one row)
• Mix of "NaN" and blank for empty fields
• "Kind Of" vs "A little" for previous lessons field
Clean these before import. A Python script can handle bulk formatting.
2

Build the Import Script

30 minutes

Write a Python script that reads the xlsx, cleans the data, and outputs a HoneyBook-ready CSV. The script should:

  • Normalize phone numbers to (XXX) XXX-XXXX format
  • Combine swimmer details into a structured notes block per contact
  • Handle the duplicate/sibling rows (merge or flag)
  • Replace NaN/None with empty strings
  • Output: honeybook-import.csv with columns: name, email, phone, address, notes
✅ HB import requirements (confirmed):
• File must be .CSV
• At minimum, an email must be included for each contact
• Column headers can be mapped during upload (HB provides a mapping UI)
• Duplicates are detected by email — existing contacts get an "updated" icon
• Failed imports generate a downloadable error report
3

Import into HoneyBook

15 minutes

Two import paths available — choose based on whether you want projects created automatically:

Method Where Creates Projects? Best For
Contacts import Clients → Contacts → Import No — contacts only Building a contact list without cluttering the pipeline
Pipeline import Pipeline → Import Yes — one project per contact Loading active/returning clients who need projects
Recommendation: Use Contacts import for the initial bulk load. These are existing/past clients — they don't need new pipeline projects. New inquiries from the custom form will create projects via Zapier. Don't pollute the pipeline with 112 historical records.

Workstream 2
Form → HoneyBook Linkage

Wire the custom intake form to HoneyBook via Zapier. One zap, one direction, only qualified leads.

1

Architecture Recap

1
Custom form → Client-side routing (JS). Unqualified leads get declined inline. Never reach Zapier.
2
Qualified lead → Form POSTs JSON to Zapier webhook URL
3
Zapier → Catches webhook → Creates new client in HoneyBook
4
HoneyBook → New client triggers automation → Welcome email sequence fires
🚨 Critical limitation discovered: HoneyBook's Zapier integration has no "Create Project" action. The only available action is "Create Client". This means Zapier can create a contact, but NOT a pipeline project. To get a project created, you need a second step: either use HoneyBook's automation to auto-create a project when a new client is added, or have Amanda manually create the project from the new contact.
2

Zapier Setup — Step by Step

30 minutes

Prerequisite: Amanda needs a Zapier account (free tier: 100 tasks/month — more than enough for her volume).

Zap #1: Qualified Lead → HoneyBook Client TRIGGER: App: Webhooks by Zapier Event: Catch Hook → Generates a unique webhook URL → Paste this URL into sunshine-intake.html as ZAPIER_WEBHOOK_URL ACTION: App: HoneyBook Event: Create Client Field mapping: Client Name ← parent_name Email ← email Phone ← phone Address ← address Notes ← (build from all form fields — see template below) Notes field template: "QUALIFIED LEAD — {{submitted_at}} Swimmer: {{swimmer_name}} ({{swimmer_age}}) Comfort level: {{comfort_level}} Previous lessons: {{previous_lessons}} Pool heated: {{pool_heated}} Location: {{location}} Lesson location: {{lesson_location}} Medical: {{medical}} Injuries: {{injuries}} Additional swimmers: {{additional_swimmers}} Preferred times: {{preferred_times}} Referral source: {{referral_source}} Other notes: {{anything_else}}"
Zap #2: Nonprofit Inquiry → HoneyBook Client (separate zap or filter) Same webhook URL — use Zapier's Filter step: Only continue if: lead_type = "nonprofit" ACTION: App: HoneyBook Event: Create Client Notes field prefix: "NONPROFIT INQUIRY — {{submitted_at}}"
⚠️ Alternative approach — single zap with Paths: Instead of two zaps, use Zapier Paths (requires Starter plan, $20/mo) to branch: if lead_type = "qualified" → path A, if lead_type = "nonprofit" → path B. Same webhook, different note formatting. Saves a zap slot on the free tier.

Workstream 3
Communications & Texting

Amanda texts her clients constantly. What can HoneyBook actually do — and what can't it?

1

HoneyBook SMS — What It Actually Does

After thorough research, here's the reality of HoneyBook's SMS capabilities:

Feature Available? Details
Session reminders via SMS Yes Auto-sends 24hr and 1hr before sessions. Configurable in Scheduler settings.
Payment reminders via SMS Yes Auto-sends when invoices are due or overdue.
Clients can reply to SMS No Clients CANNOT reply to HoneyBook SMS. They get an error message telling them to email the business. Sent from an unattended number.
Custom/ad-hoc text messages No No way to send arbitrary texts to clients from within HoneyBook.
Two-way text conversation No Not available at all. HoneyBook SMS is one-way, automated reminders only.
Weekend/holiday sends Partial Some SMS reminders won't send on weekends/holidays — delayed to next business day. Session reminders still go out.
Client opt-in required Yes Clients must opt in when filling out forms/scheduling. Required by telecom regulations.
2

The Texting Gap — Amanda's Real Workflow

Amanda's current communication pattern (from our discovery call):

  • Rain cancellations → texts clients directly from her phone ("It's raining, we'll reschedule")
  • Running late → texts "On my way, be there in 10!"
  • Schedule changes → texts to confirm/move lesson times
  • Quick check-ins → "How did Sammy do in the bath this week?"
  • Initial contact → calls new leads after reviewing their form submission
🚨 HoneyBook does NOT replace Amanda's personal texting. Its SMS is limited to automated session/payment reminders from an unattended number. All of Amanda's real-time, conversational texting stays on her personal phone. This is actually fine — her personal touch IS her brand. But we need to set expectations clearly: HoneyBook automates the reminders so she doesn't have to remember. It doesn't replace her phone.
3

Email Deliverability Warning

Multiple reviewers across G2, Capterra, and GetApp report HoneyBook emails landing in spam. This could undermine the welcome sequence.

  • Mitigation 1: Amanda should tell clients to check spam/promotions during her initial phone call
  • Mitigation 2: Test the welcome email flow with multiple email providers (Gmail, Yahoo, Outlook) during the prototype phase
  • Mitigation 3: If deliverability is poor, we can move email to a dedicated sender (Flodesk, Mailchimp) triggered by Zapier — but this adds stack complexity

Zapier Free Tier Limits

Zapier free plan: 100 tasks/month, 5 single-step zaps. At Amanda's peak-season volume (maybe 10–20 new inquiries/week), she'd use ~40–80 tasks/month. This works on the free tier during normal operation. But:

  • If we use Paths (for qualified vs. nonprofit routing), that requires the Starter plan ($20/mo)
  • Alternative: two separate zaps with Filters (free tier compatible, uses 2 of 5 zap slots)
  • If tasks exceed 100/month (unlikely at her volume), Starter plan adds 750 tasks
Recommendation: Start on Zapier free tier with two simple zaps (one for qualified, one for nonprofit) using Filter steps. Upgrade only if volume demands it. Keep Amanda's monthly costs at $49 (HoneyBook) + $0 (Zapier) = $49/mo total.
